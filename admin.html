新增项目：推荐流程

  1. 准备图片：在仓库本地目录放入图片，例如 assets/img/你的项目名/ 或 assets/img/。文件名建议英文、无空格。
  2. 打开本管理页：部署后使用 https://你的域名/admin.html；本地开发请用本地服务器打开（勿用 file://），否则图片预览可能不准。
  3. 加载数据：在「文件」中点击「从站点加载 site-data.json」。
  4. 写入项目数据：
    • 在「原始 JSON」中，于 projects 下增加新键（项目 ID，如 my-new-game），可复制现有项目再改；至少包含 title，按需填写 images、videos、videosAfterImages 等。
    • 若需图文与 Vimeo 穿插，使用 blocks 与 vimeoEmbedOptions（可参考 squid-game-unleashed）。
  5. 首页入口：在「首页作品」增加一条：项目 ID（与 projects 的键一致）、标题、缩略图路径、alt。
  6. 检查：在「项目详情」中选中新项目，核对标题、备注、图片列表、YouTube、内容块。
  7. 导出：「文件」→「下载 site-data.json」，用其覆盖仓库根目录的 site-data.json。

提交到 GitHub

在仓库根目录执行：

git add site-data.json assets/img/
git commit -m "Add project: my-new-game"
git push

等待 GitHub Pages 更新后，首页会出现新卡片，点击进入即可查看新项目。

注意

  • 静态站没有服务器上传：图片必须进 Git 仓库后再推送；此处只填相对路径（如 assets/img/foo.png）。
  • 项目 ID 在站内须唯一，且「首页作品」里的 ID 与 projects 中的键必须一致。
